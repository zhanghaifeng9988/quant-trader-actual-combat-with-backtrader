## 完成的工作

### 1. 环境配置
- 统一使用 conda 管理环境（`environment.yml`）
- 敏感信息（API_KEY）放入 `.env`，不提交 Git
- 全局配置（路径、回测参数、筛选阈值）放入 `config.py`

### 2. 数据获取模块（`data/fetcher.py`）
- 使用 tickflow API 获取 A 股日 K 线数据
- 关键参数：`count=10000`（防默认截断）、`adjust='forward'`（前复权）
- 保留字段：`trade_date, symbol, name, open, high, low, close, volume, amount`
- 支持单只/批量下载，CSV 存储格式

### 3. 数据质量检查模块（`data/quality.py`）
- **原则**：不随意删除股票，发现问题标记出来供人工辨认
- symbol/name 为空 → 标记待人工确认
- 价格/成交量空值 → 向前填充 + 向后填充 → 回写 CSV
- **停牌日（volume=0）→ 删除整行**（Backtrader 自动跳过该日期，不产生假信号）
- 数据行数不足检测（< 60 天）
- 输出结构化质量报告

### 4. 股票池筛选模块（`data/universe.py`）

#### 预定义股票池
- **BLUE_CHIP**：107 只 A 股蓝筹，覆盖 15+ 行业
  - 传统行业：金融、食品饮料、家电、医药、能源、钢铁、化工
  - 新兴赛道：AI（光模块/芯片/算力）、机器人、航空航天、碳减排
  - 互联网、矿业、交通运输、农畜牧

## 遇到的坑与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| tickflow 数据截断 | `count` 默认 100，不管时间范围 | 显式设 `count=10000` |
| 子目录脚本导入失败 | `data/fetcher.py` 找不到 `config` | `sys.path.insert(0, str(Path(__file__).resolve().parent.parent))` |
| API 限流（10次/分钟） | delay=0.3 太快，大量失败 | delay=7 秒 + 分批下载（每批 40 只） |
| CSV 缺少日期列 | tickflow 返回 `trade_date` 而非 `date` | `keep_cols` 改为 `trade_date` |
| 成交量过滤太严格 | 100 万股/天把高价蓝筹都剔除了 | 改为成交金额过滤（> 1 亿元/天） |
| 601989.SH name 为空 | tickflow 返回的 name 字段缺失 | 手动填充 `df['name'].fillna('中国重工')` |
| 停牌数据影响回测 | volume=0 + OHLC 相同价格导致假信号、假成交 | 删除停牌日整行，Backtrader 自动跳过 |
| 成交量 vs 成交金额 | 高价蓝筹股数少但金额大，成交量过滤误杀 | 改用日均成交金额 > 1 亿元 |
